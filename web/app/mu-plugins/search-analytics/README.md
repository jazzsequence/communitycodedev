# Search Analytics

A must-use plugin for Community + Code that logs anonymous visitor searches to a custom database table and surfaces actionable insights in the WordPress admin.

## What it does

- Tracks anonymous search queries from two sources: standard WordPress search page loads (`?s=query`) and the ElasticPress Instant Results panel (via a proxy endpoint that replaces the direct browser→EP.io request)
- Stores each search event with the search term, result count, timestamp, country, user agent, and referrer
- Provides a Dashboard admin page with charts, top terms, daily volume, suggested tags, and a searchable recent searches log
- Suggests new post tags based on frequently searched terms that have no matching tag in the taxonomy, with optional AI-assisted slug normalization
- Filters out logged-in users, bots, and WordPress's own HTTP client so only genuine anonymous visitor searches are counted

## Admin dashboard

Navigate to **Dashboard → Search Analytics**.

The page is split into sections:

| Section | What it shows |
|---|---|
| Summary stats | Total searches and unique terms for the selected period |
| Daily Search Volume | Line chart of daily search counts |
| Top Search Terms | Horizontal bar chart of the 25 most searched terms |
| Daily Volume | Table of per-day search counts |
| Suggested Tags | Frequently searched terms with no matching tag, with candidate posts and a one-click "Create & apply tag" button |
| Recent Searches | Individual search events with date, country, user agent, referrer, and a CSV export link |

### Period selector

The **7 days / 30 days / 90 days / All time** buttons filter all sections to that window. The default period is configurable per-user via Screen Options.

### Screen Options

Open the **Screen Options** panel (top-right tab) to configure:

- **Recent Searches** — number of rows to display (default: 50)
- **Suggested Tags** — minimum number of searches required to surface a term as a suggestion (default: 5)
- **Default Period** — which period is pre-selected when you open the page

### CSV export

Click **Download CSV** in the Recent Searches heading to download all recorded searches (full history, not bounded by the period selector) as a `.csv` file.

## WP-CLI

### List suggested tags

```
wp cc-analytics tag-gaps [--min-count=<n>] [--days=<n>] [--apply] [--dry-run] [--format=<format>]
```

Compares frequently searched terms against the `post_tag` taxonomy and lists terms that have no matching tag and at least one candidate post (found via ElasticPress).

**Options**

| Option | Default | Description |
|---|---|---|
| `--min-count=<n>` | `5` | Minimum search count to include a term |
| `--days=<n>` | `30` | Look-back window in days. `0` = all time |
| `--dry-run` | — | Preview only, no changes made (default behaviour without `--apply`) |
| `--apply` | — | Create the suggested tags and apply them to EP-matched posts |
| `--format=<format>` | `table` | Output format: `table`, `json`, or `csv` |

**Examples**

```bash
# Preview suggestions for the last 30 days (minimum 5 searches)
wp cc-analytics tag-gaps

# Widen the window and lower the threshold
wp cc-analytics tag-gaps --min-count=2 --days=90

# Create tags and apply them to matching posts
wp cc-analytics tag-gaps --apply

# Export as JSON
wp cc-analytics tag-gaps --format=json
```

## AI integration

When the **WordPress AI plugin** (`ai`) is installed and configured with valid credentials, suggested tag slugs are generated by the AI model rather than via `sanitize_title()`. The AI groups synonyms and variants (e.g. "wordpress", "WP", "WordPress core") under a single canonical slug.

The AI integration is entirely optional. The plugin detects availability at runtime and falls back gracefully to basic slug generation when the AI plugin is absent or unconfigured.

## Database schema

The plugin creates a single table on first activation (or on schema version upgrade):

```sql
CREATE TABLE {prefix}cc_search_analytics (
    id          BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    term        VARCHAR(200)    NOT NULL,
    results     MEDIUMINT UNSIGNED NOT NULL DEFAULT 0,
    searched_at DATETIME NOT NULL,
    country     VARCHAR(2)      NOT NULL DEFAULT '',
    user_agent  VARCHAR(500)    NOT NULL DEFAULT '',
    referrer    VARCHAR(500)    NOT NULL DEFAULT '',
    PRIMARY KEY (id),
    KEY term (term(50)),
    KEY searched_at (searched_at)
);
```

Rows older than 90 days are pruned automatically by a daily WP-Cron event (`cc_search_analytics_cleanup`).

## Technical notes

**Country detection** — Uses Cloudflare's `CF-IPCountry` header when available. If Pantheon's GCDN does not forward it, falls back to a lightweight lookup against `ipinfo.io` using the client IP from `REMOTE_ADDR`. Results are cached as WordPress transients (24-hour TTL) to avoid repeated API calls from the same IP.

**EP Instant Results proxy** — Pantheon's MU plugin routes EP Instant Results searches directly from the browser to a hosted ElasticPress.io endpoint, bypassing WordPress PHP. This plugin intercepts at the `ep_instant_results_search_endpoint` filter (priority 20, after Pantheon's filter at 10) and redirects EP's `apiEndpoint` to `/wp-json/community-code/v1/ep-search`. The proxy endpoint logs the search term and forwards the request to EP.io, returning the response transparently.

**Bot filtering** — Searches are discarded when the user agent contains any of: `bot`, `crawl`, `spider`, `slurp`, `mediapartners`, `wget`, `curl`, `WordPress/` (the last one catches WordPress's own HTTP client used by loopback/cron requests).
